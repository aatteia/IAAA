# Development Plans

This folder contains saved plans for significant development work.

## Purpose

When making changes to the portfolio that involve multiple files or require careful sequencing, save the plan here before executing. This provides:

- A reference if the work session is interrupted
- Context for future sessions with Claude
- A record of what was intended vs what was implemented

## Naming Convention

Use descriptive filenames with dates:
- `2026-02-feature-name.md`
- `2026-02-bug-fix-description.md`

## Template

```markdown
# Plan: [Feature/Change Name]

**Date:** YYYY-MM-DD
**Status:** Draft | In Progress | Complete | Abandoned

## Goal

[What this change aims to achieve]

## Approach

[How the change will be implemented]

## Files Affected

- [ ] file1.html
- [ ] file2.css

## Acceptance Criteria

- [ ] Criterion 1
- [ ] Criterion 2

## Notes

[Any additional context or considerations]
```
