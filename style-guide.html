Common Colour Pairings

These are the tested foreground/background combinations used across the site. All pairings meet WCAG AA contrast requirements (4.5:1 for normal text, 3:1 for large text).

Body text on white

--color-grey-900 on --color-white — 12.6:1

Secondary text on white

--color-grey-700 on --color-white — 5.9:1

White on primary

--color-white on --color-primary — 6.4:1

White on gradient

--color-white on primary gradient — 6.4:1+

Body text on grey-50

--color-grey-900 on --color-grey-50 — 11.4:1

Footer text on grey-100

--color-grey-700 on --color-grey-100 — 5.4:1

Avoid: Do not pair --color-grey-500 with --color-white for body text (3.2:1 — fails AA). Use --color-grey-700 or darker for readable text.

Transitions

Use fast for micro-interactions (icon rotations, toggle states). Use base as the default for hover states on links, buttons, and nav items. Use slow for card hover lifts and larger visual changes. Use all as a shorthand when transitioning multiple properties at once (e.g. buttons).

--transition-fast — 0.15s ease
--transition-base — 0.2s ease
--transition-slow — 0.3s ease
--transition-all — all 0.2s ease

Scroll Animation

Elements with .card, .capability-card, .testimonial-card, .risk-item, .credibility-item, and .client-logo-wrapper fade in on scroll using IntersectionObserver. Animation: opacity 0→1 + translateY(20px→0) over 0.5s with staggered delays. Client logos stagger at 80ms intervals. Respects prefers-reduced-motion.

Breakpoints

The design system uses a mobile-first approach with max-width media queries. Token values (font sizes, container padding, section spacing) are redefined at each breakpoint in :root. Component-level layout changes (grid columns, nav visibility) are handled in components.css.

Tablet — max-width: 991px Font sizes reduce, section padding shrinks, 3-col grids become 2-col
Mobile — max-width: 767px Desktop nav hides, mobile menu appears, grids stack to 1-col, H1 reduces to 32px
Small mobile — max-width: 479px Buttons go full-width, client logos stack to 1-col

.mobile-menu-toggle — hamburger icon, visible below 767px. .mobile-menu — full-screen overlay with vertical link list

Skip Link

.skip-link — hidden until focused via Tab key, jumps to #main

The skip link is visually hidden off-screen and appears when a keyboard user presses Tab. It links to #main on every page. Try pressing Tab at the top of any page to see it.

Used in: All pages (index.html, style-guide.html, privacy.html). Required for WCAG keyboard navigation compliance.
